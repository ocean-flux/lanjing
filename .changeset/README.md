# Changesets Governance

This repo uses Changesets to manage version intent for the Lanjing app.

- `main` is the stable integration branch.
- `beta` is the prerelease integration branch.
- Root `package.json` is the normalized version source of truth after the version sync step.
- `package.json`, `src-tauri/Cargo.toml`, and `src-tauri/tauri.conf.json` are synced by repo scripts.
- `CHANGELOG.md` is generated by Changesets on stable versioning.
- `beta` prerelease versioning intentionally restores `CHANGELOG.md` so prerelease cadence does not rewrite the stable changelog.

Common commands:

- `pnpm run changeset:add`
- `pnpm run changeset:status`
- `pnpm run changeset:version:stable`
- `pnpm run changeset:version:beta`
- `pnpm run version:sync`
